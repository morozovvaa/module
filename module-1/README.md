# Модуль 1: Веб-технологии в образовании

**Продолжительность:** 18 академических часов (6 часов лекций + 12 часов практики)

## Цель модуля

Освоить современные веб-технологии для создания интерактивных образовательных материалов и приложений, которые можно использовать в учебном процессе.

## Задачи модуля

1. Изучить современные возможности HTML5 и CSS3 для создания привлекательных учебных материалов
2. Освоить JavaScript для добавления интерактивности
3. Познакомиться с фреймворками для быстрой разработки веб-приложений
4. Создать три практических проекта для использования в образовательном процессе

## Планируемые результаты обучения

После завершения модуля студенты смогут:
- Создавать адаптивные веб-страницы с учебными материалами
- Разрабатывать интерактивные элементы для вовлечения учащихся
- Создавать онлайн-тесты с автоматической проверкой
- Разрабатывать образовательные мини-игры
- Публиковать свои разработки в интернете

## Содержание модуля

### Лекция 1: Основы HTML5 и CSS3 для создания учебных материалов (2 часа)
### Лекция 2: JavaScript и интерактивность на веб-страницах (2 часа)
### Лекция 3: Создание образовательных веб-приложений с использованием фреймворков (2 часа)

### Практическая работа 1: Создание интерактивной учебной страницы (4 часа)
### Практическая работа 2: Разработка онлайн-теста с проверкой ответов (4 часа)
### Практическая работа 3: Создание образовательной веб-игры (4 часа)

## Используемые технологии

- HTML5 (семантическая разметка, мультимедиа)
- CSS3 (Flexbox, Grid, анимации, адаптивный дизайн)
- JavaScript (ES6+, DOM манипуляции, события)
- Bootstrap или Tailwind CSS (опционально)
- Vue.js или React (базовое знакомство)

## Рекомендуемая литература

1. "HTML и CSS. Разработка и дизайн веб-сайтов" - Джон Дакетт
2. "JavaScript для детей" - Ник Морган (несмотря на название, отлично подходит для начинающих)

## Критерии оценивания практических работ

- **Отлично (5):** Работа полностью выполнена, код чистый и хорошо структурирован, приложение работает без ошибок, реализованы все требования и добавлены дополнительные функции
- **Хорошо (4):** Работа выполнена, есть небольшие недочёты в коде или дизайне, все основные требования реализованы
- **Удовлетворительно (3):** Работа выполнена частично, основная функциональность работает, но есть существенные недостатки
- **Неудовлетворительно (2):** Работа не выполнена или не работает
